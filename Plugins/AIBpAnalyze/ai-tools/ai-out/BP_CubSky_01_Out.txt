一、蓝图概述
--------------------------------------------------------------------------
BP_CubSky_01 是一个功能完整的 **动态天空盒 / 昼夜循环系统**，负责在游戏运行时驱动
场景中的天空视觉效果。它集中管理以下子系统：

1. 太阳/月亮运动轨迹计算
2. 多层次雾效（近景雾、中景雾、远景雾）
3. 天空材质参数的实时驱动（SkyBox、Cubemap、大气）
4. 环境光（Sky Light）与定向光（Directional Light/Sun）的颜色/强度曲线
5. 昼夜状态机（夜晚 → 清晨 → 白天 → 黄昏 四段循环）
6. 后处理曝光（PostProcessVolume）、Bloom、轮廓材质等
7. 天空Cubemap纹理混合（水体特效辅助融合）
8. 可扩展性适配（画质分级驱动雾密度等参数）

四、核心函数/事件图分析
--------------------------------------------------------------------------

【Event BeginPlay】
  → Init Sky Effects（308节点）
    - 创建 mat_sky、mat_SkyCub 动态材质实例
    - 将 outline_black 材质注入 PostProcessVolume
    - 初始化三层雾参数（Density/Distance/SkySphereBlend）
    - 根据画质等级（GetOverallScalabilityLevel）调整雾密度
    - 调用 Init Sky Tex 初始化 Cubemap 纹理
  → 启动 Timeline_TimeOfDay（若 Use Time Of Day）

【Timeline_TimeOfDay 更新回调】
  → Update Sun Moon Rotation（26节点）
    - 读取 CurrentTime → 调用 Caculate Day State
    - 通过 MapRangeClamped + 三角函数计算太阳球面位置
    - 设置 DirectionalLight(Sun) 旋转
    - 通过 SunAnglePitch/Yaw 计算月亮方向向量
    - 触发 OnSkyTimeUpdate 委托

  → Caculate Day State（35节点）
    - 4段阈值比较（float <）确定 time_of_day_state 枚举
    - 更新 bIsDayTime 布尔值
    - 状态变化时触发 OnDayStateChange 委托

  → Update Sky Effects（157节点）
    - 根据 Day/Night、OnlyNight 标志选择材质驱动分支
    - 通过运行时曲线（RuntimeFloatCurve）计算太阳强度、光晕遮罩
    - 调用 Set Scalar/Vector Parameter Value 更新材质参数
    - 设置雾颜色（Lerp 在四时段颜色之间）、方向散射色/强度
    - 更新 SkyLight 强度（SkyLightIntensityCurve 曲线驱动）
    - 帧间隔更新（FramesIntervalForDirectionUpdate）减少每帧开销

【Custom Events（编辑器/外部调用）】
  - DayOrNight         : 强制切换昼/夜模式
  - UpdateSkyTex       : 触发天空纹理混合更新
  - InitSkyTex         : 初始化 Cubemap 纹理索引
  - water effect blend to next : 驱动水面纹理混合 Timeline
  - Event NotifySetTimeOfDay   : 外部设置具体时刻接口
  - UpdateSunColor             : 更新太阳颜色（调用 Update Sky Effects）
  - Event OnScalabilityModify  : 响应画质变更重新初始化

【输入调试】
  - Page Up 键绑定：触发时间推进测试（开发调试用）

五、性能特征
--------------------------------------------------------------------------
- EventGraph 共 487 个节点，为极大图，逻辑复杂度高
- Init Sky Effects 308 节点，Update Sky Effects 157 节点
- 使用帧间隔机制（FramesIntervalForDirectionUpdate）降低每帧更新频率
- 存在大量 Reroute Node（>100个），说明连线管理复杂
- 多处 Is Valid 校验防止空引用崩溃
- 使用 Timeline 而非 Tick 驱动主循环，对性能相对友好

六、优化建议（按优先级排序）
--------------------------------------------------------------------------

【P0 - 高优先级 · 影响稳定性/正确性】

1. 【变量命名拼写错误】
   - "Day/NigjhTime" 应为 "Day/NightTime"
   - "SatrsXScale" 应为 "StarsXScale"
   - "Caculate Day State" 应为 "Calculate Day State"
   - "Directional Night  fog"（多余空格）
   - "FarFog _Z_Height_scale"（多余空格）
   这些拼写错误会导致外部调用/搜索困难，应优先修正。

2. 【Page Up 调试输入绑定残留】
   EventGraph 中存在 "Page Up" 按键绑定节点（K2Node_InputKey_0），
   这是开发调试遗留代码，应在发布版本前移除，避免影响玩家输入。

3. 【执行控制台命令（Execute Console Command）】
   BeginPlay 中存在 Execute Console Command 节点，控制台命令在发布包中
   可能被优化掉或产生未预期行为，建议替换为对应的 C++ API 或 Blueprint 函数。

【P1 - 中高优先级 · 影响性能】

4. 【节点图过于庞大，建议拆分为子蓝图或组件】
   EventGraph 487个节点、Init Sky Effects 308个节点，单图逻辑过重。
   建议将雾效管理、天空光管理、纹理混合分别提取为独立 Blueprint Function Library
   或 Actor Component，提升可维护性并减少编译时间。

5. 【大量 Reroute Node 导致可读性差】
   统计到超过 100 个 Reroute Node，说明连线混乱，维护成本高。
   建议使用局部变量（Local Variables）代替长连线，或重构为更小粒度的函数。

6. 【Update Sky Effects 中多次重复读取同一变量】
   例如 mat_sky 被多次 Get（K2Node_VariableGet_17/18/27/6239 等），
   建议在函数入口处一次性读取到局部变量，减少变量访问开销。

7. 【帧间隔更新机制可进一步优化】
   FramesIntervalForDirectionUpdate 当前为手动计数，建议改用 UE 的
   SetTimerByFunctionName / SetActorTickInterval 等内置节流机制，
   逻辑更清晰且不易出错。

【P2 - 中优先级 · 影响可维护性/扩展性】

8. 【四时段颜色/参数采用数组而非散变量】
   目前 NearNightFogColor/NearMorningFogColor/NearDayFogColor/NearDustFogColor
   等以独立变量形式存储，共有 Near/Middle/Far × 4时段 × 颜色 = 12+ 个颜色变量。
   建议改为 Array 或 DataTable/Struct，支持设计师直接在表格中配置，减少蓝图变量数量。

9. 【天空纹理数组（SkyCubDiffArray/SkyCubMaskArray）与编辑器版本分离】
   存在 SkyCubDiffArray_Editor 和 SkyCubDiffArray 两套，建议通过
   WITH_EDITOR 宏或 EditorUtility 机制统一管理，避免运行时/编辑器数据不一致。

10. 【OnDayStateChange / OnSkyTimeUpdate 委托未见统一注释】
    这两个对外委托是外部系统（天气、NPC AI、水面等）的重要接口，
    建议补充详细注释说明触发时机和传递参数含义，方便多人协作。

11. 【bAreaSwitch 功能未明确文档化】
    变量 bAreaSwitch 在 Update Sky Effects 中用于分支选择，但命名语义不清晰，
    建议重命名并补充注释，说明"区域切换"的具体业务含义。

【P3 - 低优先级 · 代码质量】

12. 【"Heigh Fog" 命名不规范】
    应为 "HeightFog"，当前命名缺少 "t"，建议统一修正。

13. 【PauseSkyLight / StopSunLightUpdate 建议合并或文档说明差异】
    两个"暂停"标志功能相近，建议明确各自的作用范围差异，避免维护混乱。

14. 【Timeline_WaterTexBlend 与主天空逻辑耦合】
    水体纹理混合逻辑与天空盒逻辑放在同一蓝图中，职责不单一。
    建议将水体 Timeline 相关逻辑迁移到专用的水体 Actor 或 Component 中。

--------------------------------------------------------------------------
七、总结
--------------------------------------------------------------------------
BP_CubSky_01 是一个功能丰富、驱动能力强的天空盒系统，支持完整的四段昼夜循环、
多层雾效、动态光照和纹理混合，对游戏氛围表现有重要贡献。

主要风险点：
- 蓝图体量过大（累计近千节点），编译和维护成本高
- 多处命名错误和调试代码残留，应在版本发布前清理
- 参数散变量过多，建议数据驱动化以提升可配置性

建议优先处理 P0 问题（命名/调试代码），再逐步推进 P1 重构。
