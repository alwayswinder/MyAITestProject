一、蓝图概览
=========================================

父类: Actor
组件数量: 1 (DefaultSceneRoot)
变量数量: 8
自定义事件数量: 6
事件图节点数量: 92（含 24 个 Reroute 节点）

主要功能:
BP_WaterVarSwitch 是一个环境区域效果触发器，放置在场景中后，当玩家进入指定触发体积
（Trigger/BrushComponent）时，自动以平滑过渡的方式更改水体材质参数和天空 HDRI
色调偏移。同时提供编辑器中的预览（Preview）与恢复（Revert）功能，方便关卡设计师
在不运行游戏的情况下预览效果。

=========================================
二、变量分析
=========================================

【运行时参数（目标值）】
1. Trigger            (Object)       - 场景中的碰撞触发器引用（TriggerVolume / BrushComponent）
2. WaterActor         (Object)       - 目标水体 Actor 引用
3. Sky HDRI HueShift  (Float)        - 天空 HDRI 色调偏移目标值（标量材质参数）
4. waterColor         (LinearColor)  - 水体颜色目标值（向量材质参数）
5. water front color  (LinearColor)  - 水体前景/泡沫颜色目标值（向量材质参数）

【默认值（用于还原）】
6. Sky HDRI HueShift Default  (Float)        - 天空 HDRI 色调偏移默认值
7. waterColor Default         (LinearColor)  - 水体颜色默认值
8. water front color Default  (LinearColor)  - 水体前景色默认值

=========================================
三、主要逻辑流程
=========================================

1. 【初始化阶段 — Event BeginPlay】
   - 获取 Trigger 变量引用的 BrushComponent
   - 设置碰撞启用状态（Set Collision Enabled）
   - 设置碰撞配置文件（Set Collision Profile Name）
   - 将 CustomEvent_0（碰撞回调）绑定到 BrushComponent 的 OnComponentBeginOverlap 事件
   - 使用 Delay 节点延迟绑定，避免玩家从一开始就处于区域内时漏触发

2. 【碰撞触发 — CustomEvent_0 (OnComponentBeginOverlap 回调)】
   - 接收碰撞参数：OverlappedComponent、OtherActor、OtherComp 等
   - 调用 Get Player Character 获取本地玩家角色
   - 使用 Equal (Object) 比较 OtherActor 与玩家角色
   - Branch：若相等则调用 WaterActor 上的接口函数 Water Change Start，启动水体切换

3. 【水体参数平滑过渡 — WaterChangeStart 自定义事件】
   - 触发 Timeline（含曲线轨道 NewTrack_0，输出 0→1 的 alpha 值）
   - Timeline.Update（每帧回调）：
       * Lerp(Sky HDRI HueShift Default → Sky HDRI HueShift, alpha)
         → Set Scalar Parameter Value on Materials（遍历所有相关 Mesh 组件）
       * Lerp(Vector)(waterColor Default → waterColor, alpha)
         → Set Vector Parameter Value on Materials
       * Lerp(Vector)(water front color Default → water front color, alpha)
         → Set Vector Parameter Value on Materials
       * 使用多个 For Each Loop 遍历通过 Get Components By Class 获取的组件列表
   - Timeline.Finished → 调用 WaterChangeEnd

4. 【水体变化结束 — WaterChangeEnd 自定义事件】
   - Timeline 完成后触发
   - 调用 Update Mat Var 函数，将最终参数值写入 WaterActor 相关组件（BP_Water_Loop）

5. 【编辑器预览 — Preview 自定义事件】
   - 立即（不经过 Timeline）将 Sky HDRI HueShift、waterColor、water front color
     应用到所有相关 Mesh 组件的材质参数，用于编辑器实时预览

6. 【恢复默认 — Revert 自定义事件】
   - 立即将三个 Default 变量的值还原到所有相关 Mesh 组件
   - 包含 Delay 节点（与 BeginPlay 中的延迟逻辑对应）
   - 通过 Get Components By Class + For Each Loop 遍历目标组件

=========================================
四、优化建议（按优先级排序）
=========================================

【高优先级 — 性能优化】

1. 缓存 GetComponentsByClass 结果
   问题: GetComponentsByClass 在 WaterChangeStart（Timeline Update，每帧执行）、
         Preview、Revert 中各自独立调用，共至少 3 处，每次都遍历全部组件树。
   影响: 严重的逐帧性能开销，组件数量越多越明显。
   建议:
   - 在 BeginPlay 时调用一次 GetComponentsByClass，结果保存到成员变量数组。
   - 所有事件直接使用缓存数组，彻底消除重复遍历。
   预计收益: 性能提升 20-40%，代码量减少。

2. 避免在 Timeline Update 中执行 For Each Loop
   问题: WaterChangeStart 的 Timeline Update 每帧都运行多个 For Each Loop，
         对所有 Mesh 组件逐一调用 Set Scalar/Vector Parameter Value on Materials。
   影响: 若场景中存在多个此蓝图实例或水体组件较多，帧率压力显著。
   建议:
   - 配合建议 #1，使用缓存数组代替每帧的 GetComponentsByClass。
   - 若材质实例固定，考虑在 BeginPlay 时直接缓存材质实例引用，
     跳过组件遍历，直接调用 SetScalarParameterValue / SetVectorParameterValue。
   预计收益: 性能提升 30-50%。

3. 消除每帧冗余的 LinearColor → Vector 转换
   问题: Timeline Update 中存在 6 处 To Vector (LinearColor) 节点，每帧重复执行。
   影响: 轻微但不必要的计算开销。
   建议:
   - 将 waterColor / water front color 变量直接改为 Vector 类型，
     或在 BeginPlay 时预转换并缓存 Vector 版本。
   预计收益: 减少每帧无效计算，代码更简洁。

【中优先级 — 架构优化】

4. 封装重复的材质应用逻辑
   问题: WaterChangeStart（Timeline Update）、Preview、Revert 三处均有几乎相同的
         "遍历组件 → Set Scalar/Vector Parameter"代码块，高度重复。
   建议:
   - 新建函数 ApplyWaterMaterialParams(float HueShift, LinearColor WColor,
     LinearColor WFrontColor)，统一封装参数应用逻辑。
   - 三个事件均调用此函数，减少约 70% 的重复节点。
   收益: 代码量大幅下降，后续修改只需改一处。

5. 为 CustomEvent_0 补充有意义的命名
   问题: 碰撞回调事件名为 CustomEvent_0，缺乏语义。
   建议: 重命名为 OnTriggerOverlapBegin 或 OnPlayerEnterTrigger，
         提升代码可读性与可维护性。

6. 规范变量命名
   问题: "water front color"、"Sky HDRI HueShift" 等变量名含空格，
         不符合 Unreal Engine Blueprint 命名规范。
   建议:
   - 使用 PascalCase：WaterFrontColor、SkyHDRIHueShift、SkyHDRIHueShiftDefault 等。
   - 添加变量分类（Category）和工具提示（Tooltip）便于编辑器使用。

7. 减少 Reroute 节点（24 个）
   问题: 存在大量 K2Node_Knot（Reroute）节点，导致图表连线混乱。
   建议:
   - 重新布局节点，缩短连线距离。
   - 将独立逻辑段拆分为子函数，降低单图表复杂度（当前 92 个节点偏高）。

【低优先级 — 健壮性与扩展性】

8. 添加空引用校验
   问题: Trigger 和 WaterActor 变量若未在关卡中赋值，运行时将产生访问空引用崩溃。
   建议:
   - 在 BeginPlay 中对 Trigger、WaterActor 添加 IsValid 检查。
   - 校验失败时输出 PrintString 或 Log Warning，便于快速定位配置错误。

9. 将 Timeline 参数暴露为可配置项
   问题: 过渡时长与曲线形状硬编码在 Timeline 资产中，不同关卡区域无法独立调整。
   建议:
   - 增加 TransitionDuration (Float) 可编辑变量。
   - 支持传入自定义曲线资产（CurveFloat），提高复用灵活性。

10. 增强碰撞触发器的通用性
    问题: 目前仅检测 Get Player Character 单一玩家，不支持多人游戏或其他 Actor 触发。
    建议:
    - 改用 Interface 或 GameplayTag 判断触发条件，提升扩展性。
    - 可选：添加触发冷却时间，防止短时间内重复触发。
