# 项目分析规则

## 总则

本规则适用于AI助手对本项目的分析、开发和维护工作。所有AI助手在处理本项目时必须严格遵循以下规则。

## 项目基本信息

- **项目名称**：MyAITestProject（贪吃蛇游戏）
- **项目类型**：Unreal Engine 5 项目
- **主要语言**：C++ + 蓝图
- **文档位置**：`.docs/` 目录

## 分析流程

### 第一步：了解项目结构

1. 先阅读 `.docs/项目说明.md`，了解项目概述、技术栈和游戏流程
2. 查看 `.docs/classes/` 目录下的类文档，了解核心类的功能和接口
3. 查看项目源码目录结构（`Source/MyAITestProject/`）

### 第二步：定位问题区域

根据用户需求，使用以下方法定位相关代码：
- 按类名查找头文件和源文件（如 `Snake.h` 和 `Snake.cpp`）
- 使用类文档了解类的职责和主要函数
- 参考 `项目说明.md` 中的游戏流程理解模块间关系

### 第三步：代码修改原则

1. **先理解后修改**：在修改代码前，必须充分理解现有代码的逻辑
2. **保持风格一致**：遵循项目现有代码风格和命名规范
3. **最小改动原则**：只修改与任务相关的代码，不引入无关变更
4. **注释和文档**：重要逻辑变更需要更新 `.docs/` 目录下的相关文档

## 核心类职责速查

| 类名 | 职责 | 文件位置 |

## 常见任务指导

### 1. 修复Bug

- 首先理解Bug现象，确定涉及的类
- 查阅对应类的文档，了解预期行为
- 查看相关源码，定位问题
- 修复问题后，更新相关文档（如适用）

### 2. 添加新功能

- 参考 `项目说明.md` 中的"扩展功能建议"
- 确定功能涉及的模块
- 遵循现有代码模式实现
- 更新 `.docs/` 目录下的相关文档

### 3. 优化代码

- 先分析性能瓶颈
- 优化时保持功能不变
- 遵循最小改动原则
- 确保优化后的代码可维护

## 文档更新规则

- 类的功能、属性或接口发生变化时，必须更新 `.docs/classes/` 下的对应文档
- 项目架构或流程发生重大变化时，必须更新 `项目说明.md`
- 文档使用中文编写，结构清晰

## 注意事项

1. 不要随意修改 `.docs/` 目录下的现有文档，除非确实需要更新
2. 修改代码前，确保理解Unreal Engine的相关机制
3. 涉及蓝图的修改，需参考 `项目说明.md` 中的"蓝图使用"部分
4. 保持项目的可扩展性，遵循面向对象设计原则
